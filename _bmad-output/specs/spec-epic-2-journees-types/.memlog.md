---
topic: Epic 2 - Les Journées Types
updated: 2026-07-04T12:41
---

- (decision) Pourquoi : Structurer le temps via des routines réutilisables (Journées Types) pour réduire la charge mentale de planification hebdomadaire.
- (capability) Création de modèles (CAP-1) : L'utilisateur peut créer des Journées types (ex: Télétravail) contenant des plages horaires liées à des catégories. Succès : Les plages s'affichent comme conteneurs visuels colorés par catégorie.
- (capability) Planification (CAP-2) : Assigner des Journées types à des dates du calendrier. Succès : Le système génère la structure de la journée en base locale pour ces dates exactes.
- (capability) Duplication (CAP-3) : Dupliquer l'organisation d'une semaine sur la semaine suivante. Succès : Les journées types sont copiées en un seul clic.
- (capability) Peuplement Initial / Onboarding (CAP-4) : Afficher une journée type immédiatement à la première ouverture. Succès : Un modèle 'Journée Standard' est généré silencieusement et assigné à la date du jour, visible dans l'onglet 'Aujourd'hui'.
- (constraint) Stockage Local : Aucune génération infinie ou automatique de jours dans la base (AR4). La structure n'est créée en IndexedDB qu'à l'assignation.
- (constraint) Interface Zen : Cibles tactiles >= 44x44px, pas de rouge d'alerte, contour focus clavier obligatoire, UI conteneurs visuels colorés par catégorie.
- (constraint) Résilience DB : Toute mutation Dexie.js (ex: création de journée) doit être enveloppée dans un try/catch avec gestion de l'état isSubmitting pour éviter les doublons.
- (non-goal) Pas d'intégration technique en lecture/écriture avec des agendas natifs (Google Calendar, etc.).
- (non-goal) Pas de vue de synthèse hebdomadaire complète (réservé à l'Epic 4).
- (non-goal) Le Glisser-déposer de tâches de l'Inbox vers les plages n'est pas couvert ici (réservé à l'Epic 3).
- (event) Pass 1 - Coherence: Validé.
- (event) Pass 2 - Preservation: Validé. Toutes les FR et UX-DR de l'Epic 2 ont été couvertes.
