- (decision) Mode de travail: Coaching path. Pas de document alternatif demandé, seul le Spine sera produit.
- (decision) Stack technique validée: Local-First SPA avec React, Vite, Vanilla CSS, Dexie.js (IndexedDB) et Vite-PWA.
- (decision) Affectation des journées: Planification manuelle semaine par semaine avec possibilité de dupliquer une semaine précédente (Option B).
- (constraint) Si un 'Saut de cycle' échoue (aucune plage de la bonne catégorie dans les jours planifiés), la tâche retourne dans le Dépôt (Inbox).
- (event) spine finalized
- (question) L'utilisateur demande les options d'hébergement pour déployer et récupérer la webapp PWA.
- (decision) Déploiement et Hébergement : Utilisation d'un hébergeur statique gratuit (Vercel, Netlify ou GitHub Pages) puisque l'application n'a pas de backend. L'installation PWA se fera via le navigateur mobile.
- (decision) Déploiement et Hébergement : Choix définitif de GitHub Pages (associé à GitHub Actions) puisque le projet sera open-source sur GitHub, rendant l'utilisation gratuite et l'intégration parfaitement centralisée.
